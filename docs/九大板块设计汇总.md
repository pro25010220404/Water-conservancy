## 一、系统总览

### 1.1 项目定位

基于 **LSTM 时序预测** + **DQN 深度强化学习** 的水电站闸门智能调度 Web 监控平台。采用**云-边-端**三层架构，前端负责云端 Web 可视化展示与管理配置，**不参与实时 AI 推理和控制指令下发**。

### 1.2 技术栈

| 类别 | 选型 |
|------|------|
| 框架 | Vue 3（`<script setup lang="ts">`） |
| 状态管理 | Pinia |
| UI 组件库 | Element Plus |
| 大屏可视化 | ECharts + DataV（`@kjgl77/datav-vue3`） |
| 3D 引擎 | Three.js |
| 类型 | TypeScript |
| 样式 | SCSS + CSS Variables（设计 Token 统一管理） |
| 请求 | Axios（统一拦截器封装） |
| 路由 | Vue Router 4 |
| 实时通信 | WebSocket（主通道）+ HTTP 轮询（降级） |
| 构建 | Vite |

### 1.3 设计主题

| 规范项 | 要求 |
|--------|------|
| 主色调 | 深色科技风：背景 `#0a1628` ~ `#0d2137`，强调色 `#00d4ff` / `#1890ff` |
| 字体 | 中文：思源黑体 / PingFang SC；数字：DIN / Roboto Mono |
| 圆角 | 卡片 4px，按钮 4px，弹窗 8px |
| 动效 | 数据刷新淡入 200ms；告警滑入 300ms；3D ≥ 30 FPS |

### 1.4 用户角色

| 角色 | role 枚举 | 核心权限 |
|------|-----------|----------|
| 值班运维人员 | `operator` | 大屏监控、告警处理、设备巡检、远程重启 |
| 调度决策工程师 | `dispatcher` | 调度审核、人工干预、急停、历史查询 |
| 站长/管理人员 | `manager` | 全局管理、报表审计、账号解锁、系统配置 |
| 系统管理员 | `admin` | 用户权限管理、系统运维、模型管理 |
| 算法工程师 | `algorithm_engineer` | 仿真训练、模型评估 |
| **所有登录用户** | — | **急停按钮**（不受角色限制） |

## 二、九大板块设计详情

### 板块一：登录页 `/login`

**定位**：安全、友好的身份认证入口，满足企业级密码策略。

**核心设计要点**：

| 维度 | 设计内容 |
|------|----------|
| **视觉背景** | Three.js 水粒子系统（15,000+ 粒子模拟水流动效）；右侧 3D 大坝线框模型缓慢旋转；鼠标视差交互；根据本地时间自动切换晨曦/正午/黄昏/深夜四种色调 |
| **表单区** | 毛玻璃半透明卡片；用户名 + 密码（可显隐切换）+ 记住密码；时段感知欢迎语 |
| **密码策略** | 长度 8-32 位 + 大写 + 小写 + 数字 + 特殊字符（5 项规则）；实时强度指示条（弱/中/强）+ checklist |
| **首次改密** | 后端返回 `must_change_password: true` 时弹出强制改密弹窗（不可关闭/不可绕过）；支持实时强度校验 |
| **账号锁定** | 连续 5 次密码错误锁定 30 分钟；锁定弹窗含倒计时；第 3/4 次失败预警提示剩余次数 |
| **过场动画** | 登录成功后 1.5 秒"穿越闸门"过渡动画（光圈扩散 → 径向模糊推进 → 粒子汇聚 → 大屏模块依次淡入） |
| **路由守卫** | 未登录 → 跳转 `/login`；已登录访问 `/login` → 跳转 `/dashboard`；无权限页面 → Toast + 跳转默认页 |
| **响应式** | ≥1024px 左右分栏；768-1023px 隐藏品牌区；<768px 降级为静态渐变背景 |

**关键接口**：`POST /api/auth/login`、`POST /api/auth/change-password`、`POST /api/auth/refresh`、`POST /api/auth/logout`

### 板块二：监控大屏 `/dashboard`

**定位**：7×24 实时态势总览，全系统核心可视化中枢。

**页面布局（1920×1080 基准）**：

```
┌─ 顶部标题栏（72px）：Logo + 系统名 + 实时时钟 + 系统状态灯 + 语音/时光机/急停/用户 ─┐
├─ 4 张 DataV 数据卡片（上游水位 / 入库流量 / 闸门开度 / 发电功率）──────────────────┤
├─ 3D 数字孪生大坝 ────┬── 设备健康面板 ────┬── 视频监控面板 ──────────────────────┤
├─ 趋势图（含 AI 预测叠加）────────────┬── GIS 流域地图 ──────────────────────────┤
├─ 告警列表（滚动播报，最近 20 条）─────────────────────────────────────────────────┤
└──────────────────────────────────────────────────────────────────────────────────┘
```

**核心功能矩阵**：

| 功能模块 | 核心设计 |
|----------|----------|
| **数据卡片** | 4 张 DataV 边框卡片；数值翻滚动画（CountTo）；同比昨日变化箭头；点击可下钻跳转详情页 |
| **3D 数字孪生** | Three.js 低面数大坝模型（≤30,000 面）；水位随数据实时涨落（半透明蓝色材质）；闸门开合 Tween 动画（600ms）；水流粒子特效（数量正比于流量）；OrbitControls 旋转/缩放/双击聚焦闸门 |
| **设备健康面板** | 传感器在线率环形图；PLC/网关/AI 引擎状态灯 + 心跳时间；异常设备滚动列表 + 巡检/排查入口 |
| **趋势图** | ECharts 双 Y 轴（水位 + 流量）；1h/6h/24h 切换；实测值实线 + AI 预测虚线 + 置信区间半透明带；预警线标注；实时追加滚动 |
| **GIS 流域地图** | ECharts map3D / MapBox GL；测站浮标 + 状态色标；水流方向动画；气象雷达叠加（可选）；告警联动闪烁 |
| **视频监控** | RTSP → HLS 转码；多路摄像头 Tab 切换；截图 + 全屏；告警联动自动切换最近摄像头 |
| **声光告警** | 紧急告警：红色脉冲闪烁 + 三连提示音；重要告警：橙色静态高亮 + 桌面通知；静音切换记忆 |
| **语音助手** | Web Speech API 中文语音指令（"显示 X 号闸门"/"过去 X 小时趋势"/"静音"等）；TTS 语音反馈 |
| **键盘快捷键** | `Ctrl+E` 急停；`1/2/3/4` 切换趋势时间范围；`M` 静音；`F` 全屏；`?` 快捷键面板 |

**数据通信策略**：
- WebSocket 优先（指数退避重连：1s→2s→4s→8s→30s）
- WS 断开 3 秒后自动降级为 HTTP 5 秒轮询
- 页面隐藏时暂停轮询，重新可见时拉取全量快照
- 心跳 ping/pong 30s/60s

**自适应方案**：`transform: scale()` 等比缩放；1920×1080 基准；最小支持 1366×768；兼容 16:9 和 16:10

### 板块四：告警管理 `/warning`



**定位**：结构化告警管理，核心原则是**瞬时超限仅记录日志，持续 ≥30s 才触发正式告警**。

**页面布局**：

```
┌─ 筛选区：告警级别▼ / 处理状态▼ / 告警类型▼ / 时间范围 / 关键词搜索 ─┐
├─ 告警表格：时间 | 级别(色标) | 类型 | 内容 | 当前值/阈值 | 持续时长 | 状态 | 操作 ─┤
├─ 分页器 ─────────────────────────────────────────────────────────────┐
├─ 弹窗层：确认弹窗 / 处置弹窗（含监测快照）──────────────────────────┘
```

**核心业务规则（30 秒防抖）**：

```
每次监测数据 → 超阈值？→ 否 → 清除计时
                       → 是 → 已在计时？→ 否 → 开始计时 + 写超限日志（不告警）
                                        → 是 → 累计 ≥ 30s？→ 否 → 继续计时
                                                              → 是 → 创建正式告警 + WS 推送
```

| 对比维度 | 超限日志（瞬时） | 正式告警 |
|----------|:--:|:--:|
| 列表展示 | 否 | 是 |
| 弹窗通知 | 否 | 是 |
| 角标计数 | 否 | 是 |
| 需处置 | 否 | 是 |

**告警级别**：紧急（红）、重要（橙）、一般（黄）

**告警类型**：高水位、低水位、流量突变、设备离线、执行超时、执行失败

**核心功能**：

| 功能 | 设计 |
|------|------|
| **筛选** | 级别/状态/类型/时间范围/关键词；筛选变更后分页重置为第 1 页 |
| **确认** | 未处理 → 弹出确认弹窗（展示级别/类型/内容/当前值/阈值/触发时间 + 监测快照）；二次确认，操作不可逆 |
| **处置** | 未处理/已确认 → 弹出处置弹窗（处置措施必填 10-500 字）；提交后状态变为"已处置" |
| **实时通知** | WS 推送新告警 → 右上角弹窗通知 + 菜单角标未处理数量 + 可选声音提示 |
| **详情展开** | 点击行展开触发时刻前后监测快照 |

### 板块五：调度决策 `/dispatch`

**定位**：AI 调度决策的可解释性展示与人工干预控制台。

**页面布局**：

```
┌─ 当前运行状态 ────────────┬─ AI 决策详情 ───────────────────────────┐
│ 运行模式：[自动|手动]      │ 推荐动作 + 预期效果                      │
│ 水位/流量/开度             │ 影响因素列表（名称/当前值/方向/权重）     │
│ 自动执行权限：L1/L2/L3     │ 方案对比表（方案A/B/C → 各指标对比）     │
│                           │ 置信度进度条（≥80%绿/60-79%黄/<60%红）  │
│                           │ [采纳建议] [忽略] [查看详情]             │
├─ LSTM 预测图（1h/3h/6h）───────────────────────────────────────────┤
├─ 三级自动执行权限配置（L1仅建议 / L2半自动 / L3全自动）────────────┤
├─ 手动干预：滑块调节开度 [执行] [急停] ──────────────────────────────┤
├─ 调度记录（最近 20 条）────────────────────────────────────────────┤
└────────────────────────────────────────────────────────────────────┘
```

**核心设计要点**：

| 模块 | 设计 |
|------|------|
| **影响因素 ★** | 入库流量、LSTM 预测趋势、防洪裕度、发电效益、生态流量 — 每项含方向箭头（↑/↓/—）和权重占比 |
| **方案对比 ★** | 2-3 个候选方案表格对比（目标开度/预期水位/发电量/安全评分/综合得分）；推荐方案行高亮 |
| **置信度 ★** | 0-100% 数值 + 颜色进度条；<60% 警告"建议人工复核后再执行" |
| **三级权限** | L1 仅建议（不自动下发）/ L2 半自动（置信度≥80%且开度变化≤10%时自动）/ L3 全自动；仅管理员/算法工程师可修改；变更需二次确认 + 操作日志 |
| **LSTM 预测图** | 1h/3h/6h 切换；水位/流量指标切换；历史实测与预测曲线衔接；"当前时刻"竖线标注；告警阈值线 |
| **手动干预** | 滑块 0-100%，步进 1%；执行二次确认；所有操作写入调度日志不可删除 |
| **急停** | 红色醒目按钮；二次确认后停止执行并进入安全模式；急停后建议切换为手动+L1 |

**核心业务规则**：开度 0-100% 范围校验；执行中禁止下发新指令（急停除外）；每次 AI 决策须保存完整快照。

### 板块六：数字孪生 `/simulation`

**定位**：虚拟仿真环境，支持场景推演与 AI 模型迭代。四项核心能力：**仿真视图、AI 模型导入训练、方案评估报告、历史故障复盘**。

**页面布局**：

```
┌─ 仿真视图区 ────────────────────┬─ Tab 切换 + 操作面板 ────────────┐
│ 蓄水池 + 闸门 + 水路             │ [仿真控制] [模型训练] [评估报告]   │
│ 水位/开度实时动画                │ [故障复盘]                        │
│ 叠加数值：水位/流量/开度/仿真时间 │                                    │
└─────────────────────────────────┴──────────────────────────────────┘
```

**四大核心能力**：

| 能力 | 设计要点 |
|------|----------|
| **仿真视图** | 2D 示意或轻量 3D 可视化；水位升降/闸门开度/流量方向随数据实时驱动；仿真时间叠加显示 |
| **仿真控制** | 预设场景（正常/洪水/枯水/暴雨）+ 自定义参数；开始/暂停/继续/重置；1x-10x 倍速；实时绘制水位流量曲线；可叠加 AI 方案 vs 基准方案对比；结束输出摘要 |
| **AI 模型训练** | 上传 .pt/.pth 模型文件；模型状态：待验证→验证中→可用/不可用；可选历史数据或 CSV 作训练集；训练进度条 + loss 曲线；训练完成可激活或下发边缘端；历史版本回滚 |
| **评估报告** | 仿真结束一键生成报告（场景/初始条件/方案/指标对比/AI vs 基准）；含水位流量曲线图；在线预览 + PDF/Markdown 下载；历史报告列表 |
| **故障复盘** | 列表展示历史故障；复盘详情含时间线还原（告警→决策→执行→处置）；关联监测曲线/调度日志/告警记录；填写根因分析/改进措施；"导入仿真"一键复现故障场景 |

**业务规则**：仿真数据与生产数据隔离；模型激活前须通过 smoke test；同一时刻仅允许一个仿真实例；故障导入仿真时自动填充 ±30min 参数。

### 板块八：系统设置 `/settings`

**定位**：平台配置管理中台（仅管理员可访问），四个 Tab 切换。

**Tab1：告警阈值配置**

| 设计要点 |
|----------|
| 可配置参数：上游水位、下游水位、入库流量、出库流量、闸门开度、发电功率（每项有下限和上限） |
| 告警防抖时间（10-120s，默认 30s） |
| 告警渠道开关：Web 弹窗/声音/短信/邮件 |
| 每行独立编辑，修改后保存按钮激活；下限 < 上限校验；恢复默认值二次确认 |
| 保存后通过 WS 实时推送至边缘端（热更新，无需重启） |

**Tab2：多目标权重配置（★ 核心难点）**

| 设计要点 |
|----------|
| 三个联动滑块：发电效益 + 防洪安全 + 生态流量 = 1.0 |
| **联动算法**：拖动滑块 A → 余量 (1-A) 按当前比例分配给 B 和 C；如果 B+C=0 则平分 |
| 预设方案：均衡(0.40/0.35/0.25)、发电优先(0.60/0.25/0.15)、安全优先(0.15/0.70/0.15)、生态优先(0.20/0.25/0.55) |
| 合计值实时显示，不等于 1.0 时标红禁止保存 |
| 保存后 WS 推送新权重至边缘端 → DQN 参数热更新 |
| 权重变更历史表格 |

**Tab3：模型管理**

| 设计要点 |
|----------|
| 拖拽上传 .pth/.onnx/.pt 模型文件；后端校验格式 |
| 模型列表：名称/类型(LSTM/DQN)/版本/大小/状态(激活中/备用)/评估指标/操作 |
| 激活：WS 下发新模型至边缘端热加载 |
| 回滚：切换到上一个激活版本 |
| 删除：仅非激活模型可删，需二次确认 |
| 详情：弹窗展示训练参数、准确率曲线缩略图 |

**Tab4：用户管理**

| 设计要点 |
|----------|
| 表格：用户名/姓名/角色/状态/最后登录/操作(编辑/重置密码/锁定解锁/删除) |
| 搜索(用户名/姓名) + 角色筛选 + 状态筛选 |
| 新增用户弹窗：用户名/密码/姓名/角色/邮箱 |
| 锁定/解锁：连续 5 次失败自动锁定 15 分钟；管理员可手动锁定或提前解锁 |
| 重置密码：生成临时密码或手动输入 → 用户下次登录强制改密 |
| 删除：二次确认，不可删除自己 |

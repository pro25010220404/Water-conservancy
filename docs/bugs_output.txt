分模块精细化测试阶段BUG汇总
一．登录界面
标题 （登陆界面密码未自动回填）
勾选“记住密码”后退出登录，返回登录页密码未自动回填
项目
水电站闸门智能调度系统
模块
登录认证 / 记住密码
复现步骤
正常登录系统并勾选“记住密码”
退出登录返回登录页
查看密码框
预期结果
退出后返回登录页时，密码应自动回填或按设计保留记住状态。
实际现象
返回登录页后，密码未自动输入，记住密码功能未生效。
严重等级
低 / Minor
业务影响
影响登录便捷性，记住密码功能不可用。
修复建议
检查记住密码的缓存保存和登录页回填逻辑。
2. 标题 （登录界面自动登录功能无效）
勾选“自动登录”后退出登录返回登录页，未自动登录
项目
水电站闸门智能调度系统
模块
登录认证 / 自动登录
复现步骤
登录时勾选“自动登录”
成功进入系统
退出登录返回登录页
观察是否自动进入系统
预期结果
勾选自动登录后，返回登录页时应自动完成登录，或按设计直接跳转系统首页。
实际现象
勾选“自动登录”后，退出返回登录页仍需手动输入账号密码，没有自动登录。
严重等级
中 / Major
业务影响
自动登录功能失效，影响登录便捷性和用户体验。
修复建议
检查自动登录的 token / 缓存保存逻辑，以及登录页自动跳转判断逻辑。
3. 标题（退出登录后的访问控制异常，退出后页面未正确拦截，路由/缓存/登录态处理异常）
退出登录后多种场景未正确拦截，内部页面可继续访问并出现 404
模块
登录认证 / 退出登录 / 路由拦截
复现步骤
正常登录系统
进入任意业务页面
退出登录后分别测试：
点击浏览器返回键（在浏览器历史记录找到刚刚打开的网页---跳转的不是登录页而是404）
直接粘贴内网页面地址访问（成功登录后复制网页地址—退出登录后再复制刚刚的网址---打开的应该是登录页而不是404）
多标签页登录后切换页面并刷新（同个账号多页面同时打开---退出登录一个页面----其他页面应该自动跳转登录页，而不是依旧可以访问网页内部）
观察页面结果
预期结果
退出登录后，所有内部页面访问都应被拦截，并统一跳转到登录页。
实际现象
退出后上述场景下仍可能进入系统内部页面，未正确跳转登录页，部分情况下出现 404 Not Found。
严重等级
严重 / Critical
业务影响
退出登录后访问控制失效
可能暴露系统内部页面
影响系统安全性和用户体验
登录态管理存在异常
修复建议
检查退出登录后的 token 清理逻辑
检查路由守卫是否生效
检查多标签页登录态同步
检查 Nginx / 前端路由回退配置
4. 标题
账号或密码输入错误时，系统提示“登录已过期，请重新登录”，提示不准确
模块
登录认证
复现步骤
进入登录页
输入错误账号或错误密码
点击登录
观察弹窗提示
预期结果
应提示“账号或密码错误”或同类登录失败信息。
实际现象
系统提示“登录已过期，请重新登录”，与实际错误原因不符。
严重等级
低 / Minor
业务影响
提示信息不准确，容易误导用户
不利于判断问题原因
影响登录体验
修复建议
区分“登录过期”和“账号密码错误”两种情况
不同错误返回不同提示文案
5. 标题
登录页按 Tab 键后焦点跳到浏览器顶部导航栏，未按页面控件顺序切换
模块
登录页 / 键盘可访问性
复现步骤
打开登录页
使用键盘按 Tab 键依次切换焦点
观察焦点顺序
预期结果
焦点应按登录页内控件顺序切换，例如： 用户名输入框 → 密码输入框 → 记住密码 → 自动登录 → 登录按钮
实际现象
按 Tab 键后，焦点未停留在页面控件上，而是跳到了浏览器上方导航区域。
严重等级
中 / Major
业务影响
键盘无法正常操作登录页
影响可访问性和表单操作体验
对触屏键盘、辅助设备用户不友好
修复建议
检查登录页控件的 tabindex 顺序
检查是否存在可聚焦元素缺失或顺序错乱
避免页面中无关元素抢占焦点
6. 标题
无网络状态下点击登录无响应，页面无错误提示
模块
登录认证
复现步骤
断开网络
打开登录页
输入账号密码
点击“登录”
观察页面反应
预期结果
无网络时应给出明确提示，例如“网络连接失败，请检查网络后重试”，并停止登录请求。
实际现象
点击登录后页面无明显反应，停留在当前登录页，没有错误提示或加载失败提示。
严重等级
中高 / Major
业务影响
用户不知道是网络问题还是系统卡住
登录失败反馈不明确
影响系统可用性和排障体验
修复建议
登录请求增加网络异常捕获
无网时显示明确提示
增加 loading 和失败状态反馈
8. 标题
综合概览卡片数据未按 30s 刷新，页面数据长时间不更新
项目
水电站闸门智能调度系统
模块
监控大屏 / 综合概览
复现步骤
打开综合概览页面
保持页面不操作
观察上游水位、入库流量、发电功率、闸门开度、库容等卡片数据
等待超过 30 秒
预期结果
卡片数据应按文档要求约 30s 自动刷新一次，展示最新监测值。
实际现象
卡片数据长时间保持不变，没有按 30s 自动更新。
严重等级
中 / Major
业务影响
页面实时性不足
用户看到的数据可能不是最新值
影响监控准确性和调度判断
与设计文档要求不一致
修复建议
检查前端定时刷新逻辑是否正常
检查 WebSocket / 轮询数据是否成功接收
检查接口返回是否有新数据
确认 30s 刷新任务是否被页面缓存或状态管理阻断
标题
综合概览页面存在功能失效与交互异常，卡片数据不刷新、面板误收起、区域误跳转
项目
水电站闸门智能调度系统
模块
监控大屏 / 综合概览
复现步骤
打开综合概览页面
观察卡片数据是否自动刷新
点击页面任意区域，查看是否会误触收起
点击 AI 模型健康度区域，观察是否会误跳转
预期结果
卡片数据应按 30s 刷新
只有点击“收起”按钮时才收起
只有点击明确入口时才跳转
实际现象
卡片数据长时间不更新
页面任意区域点击都可能触发收起
AI 模型健康度区域任意点击都会跳转
严重等级
中高 / Major
业务影响
实时监控失效
页面交互不可控
用户容易误操作
影响大屏使用体验和调度判断
修复建议
检查定时刷新和数据订阅逻辑
限制收起事件只绑定到按钮
限制跳转事件只绑定到明确入口
检查父级容器是否误绑定点击事件
什么时候建议分开写
如果你们后面要给开发排查，我建议：
功能失效类单独一条
误触/误跳转类单独一条
这样更好定位。
9. 标题
综合概览子页面打开后，左侧导航栏未选中对应模块，当前页面位置不明确
模块
综合概览 / 子页面跳转 / 左侧导航
预期结果
从综合概览进入子页面后，左侧导航栏应保持对应模块高亮，帮助用户识别当前所在位置。
实际现象
页面已进入综合概览相关子页面，但左侧导航栏未选中“综合概览”，当前模块状态不明显。
严重等级
低到中 / Minor to Major
业务影响
用户难以判断当前位置
页面层级关系不清楚
影响导航一致性和使用体验
修复建议
子页面跳转后同步左侧菜单高亮状态
确保当前路由和菜单状态一致
如果是弹窗式子页面，也应给出当前模块提示
标题
地图点位弹窗关闭按钮无实际作用，弹窗自动消失导致交互冗余
项目
水电站闸门智能调度系统
模块
水情监测 / 地图点位弹窗
复现步骤
点击地图上的圆圈点位
观察弹出的信息小窗
尝试点击右上角“×”
观察弹窗行为
预期结果
弹窗应有明确的关闭方式，关闭按钮应可手动关闭，或不显示无意义的关闭按钮。
实际现象
弹窗在未点击“×”前就会自动消失，关闭按钮存在感不强，实际作用不明显。
严重等级
低 / Minor
业务影响
交互冗余
用户容易误解关闭按钮是否有效
影响界面简洁性和易用性
修复建议
如果弹窗会自动消失，可去掉右上角“×”
如果需要手动关闭，应保证关闭按钮有效且行为稳定
优化弹窗停留时间和关闭逻辑
标题
点击左上角“四条杠”后，左侧导航栏显示位置异常，未与页面内容对齐
项目
水电站闸门智能调度系统
模块
页面布局 / 左侧导航栏折叠展开
复现步骤
打开页面
点击左上角“四条杠”按钮
观察左侧导航栏与主内容区域的对齐情况
预期结果
点击后左侧导航栏应正常展开/收起，并与页面整体布局对齐。
实际现象
点击“四条杠”后，左侧导航栏显示位置偏移，和主内容区域没有对齐，整体布局看起来不整齐。
严重等级
低 / Minor
业务影响
页面视觉效果不美观
影响界面一致性
容易让用户感觉页面布局异常
修复建议
检查折叠展开后的布局样式
检查左侧导航栏宽度和定位是否正确
检查主内容区域是否随导航栏状态同步调整
标题
双时段对比页面时间范围校验异常，非法时间段点击查询后提示数据参数错误
项目
水电站闸门智能调度系统
模块
历史查询 / 双时段对比
复现步骤
进入“双时段对比”页面
选择不符合规则的时间范围
点击“对比查询”
观察系统提示
再选择正常时间范围进行查询
观察页面结果
预期结果
时间范围应有明确限制
后面的时间不能早于前面的时间
如果时间范围非法，应在前端直接提示用户，而不是提交后再报错
正常时间范围下应可正常查询
实际现象
时间范围出错时，会弹出“数据参数错误”提示
改成正常时间范围后又可以正常显示
说明页面虽然能报错，但时间范围校验不够友好或不够明确
严重等级
中 / Major
业务影响
用户不知道具体哪里填错
容易误操作
影响时间对比查询体验
校验提示不够清晰
修复建议
增加前端时间范围校验
后一个时间点不能早于前一个时间点
错误提示改成更明确的文案，例如：
结束时间不能早于开始时间
时间范围填写有误
避免直接弹出笼统的“数据参数错误”

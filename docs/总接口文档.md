### 公共请求头

| Header | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | string | 是（除登录外） | `Bearer {token}` |
| Content-Type | string | 是（POST/PUT） | `application/json` 或 `multipart/form-data` |
| trace_id | string | 否 | 请求唯一标识，用于链路追踪 |

### 认证方式

- **JWT 认证**：除公开接口外，请求头需携带 `Authorization: Bearer {token}`
- 登录接口：`POST /api/v1/login`（account + password），返回 JWT token
- Token 刷新：`POST /api/v1/refresh`
- **EdgeToken HMAC 认证**：边缘端调用 `/v1/edge/*` 路由使用 `Authorization: EdgeToken <edge_id>.<timestamp>.<nonce>.<signature>`，签名算法：`HMAC-SHA256(edge_id + "\n" + timestamp + "\n" + nonce + "\n" + METHOD + "\n" + path + "\n" + body, api_secret)`。校验规则：时间戳偏差 > 300s 拒绝，Nonce 重复使用拒绝（缓存 1h），签名不匹配拒绝。EdgeToken 路由独立于 `auth:api`，不要求用户登录。

### 时间格式

- 所有时间字段统一为北京时间 `Y-m-d H:i:s`（Asia/Shanghai，UTC+8），无时区后缀

### 分页规范

- 参数名统一：`page`（int，默认1）、`page_size`（int，默认20）
- 返回结构统一：`{ "total": int, "list": [...] }`

## 1. 认证模块--Fmy

### 1.1 用户登录

- **接口路径：** `POST /api/auth/login`
- **请求方式：** POST
- **中间件：** 无
- **备注：** 数据来源 `users` 表、`roles` 表

**请求参数（Body JSON）**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| account | string | 是 | 登录账号，对应 `users.account` |
| password | string | 是 | 明文密码，HTTPS传输，后端bcrypt验证 |
| remember | boolean | 否 | 是否记住密码，默认 false |

**请求示例**

```json
{
    "account": "admin",
    "password": "xxxx",
    "remember": false
}
```

**返回参数**

| 字段 | 类型 | 说明 | 备注 |
|------|------|------|------|
| token | string | 登录凭证令牌 | — |
| token_expire_time | string | 令牌过期时间，datetime格式 | — |
| user_info | object | 当前登录用户基础信息 | — |
| user_info.id | bigint | 用户主键 | `users.id` |
| user_info.account | string | 登录账号 | `users.account` |
| user_info.realname | string | 用户真实姓名 | `users.realname` |
| user_info.role_code | string | 角色编码 | `roles.code` |
| user_info.role_name | string | 角色名称 | `roles.name` |

**返回示例**

```json
{
    "code": 0,
    "msg": "操作成功",
    "data": {
        "token": "eyJhbG...",
        "token_expire_time": "2026-07-03 14:30:00",
        "user_info": {
            "id": 1,
            "account": "admin",
            "realname": "管理员",
            "role_code": "admin",
            "role_name": "系统管理员"
        }
    },
    "success": true,
    "trace_id": "uuid"
}
```

**专属错误码**

| code | 说明 |
|------|------|
| 10002 | 账号/密码参数为空 |
| 20008 | 账号或密码错误 |

| 字段 | 类型 | 说明 | 备注 |

| token | string | 新的 JWT 令牌 | — |
| token_type | string | 固定值 "Bearer" | — |
| expires_in | int | 过期时间（秒） | 默认 3600（1 小时） |

**返回示例**

```json
{
    "code": 0,
    "msg": "刷新成功",
    "data": {
        "token": "eyJhbG...",
        "token_type": "Bearer",
        "expires_in": 3600
    },
    "success": true,
    "trace_id": "uuid"
}
```

### 2.4 水情检测（AI 推理）

- **请求方式：** POST
- **中间件：** 无（公开接口）
- **备注：** 调用激活的 LSTM + DQN 模型进行水情预测。传入传感器数据，返回水位预测、流量预测、闸门建议和风险评估。

**请求参数（Body JSON）**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| upstream_level | number | 是 | 上游水位（m） |
| downstream_level | number | 是 | 下游水位（m） |
| inflow | number | 是 | 入库流量（m³/s） |
| rainfall | number | 否 | 降雨量（mm/h） |
| temperature | number | 否 | 温度（℃） |
| gate1_opening | number | 否 | 闸门1开度（0~1） |
| gate2_opening | number | 否 | 闸门2开度（0~1） |
| gate3_opening | number | 否 | 闸门3开度（0~1） |

**返回参数**
| 字段 | 类型 | 说明 |
|------|------|------|
| gate_openings | array | DQN 建议闸门开度（%） |
| predicted_inflows | array | LSTM 预测 6h 入库流量 |
| predicted_levels | array | LSTM 预测 6h 水位 |
| predicted_peak_level | number | 峰值水位（m） |
| safety_flag | string | safe / warning / danger |
| decision_mode | string | L3_AUTO / L2_SUGGEST / L1_MANUAL |
| models_used | object | 本次推理使用的 LSTM + DQN 模型信息 |

**专属错误码：** 90003 模型加载失败
